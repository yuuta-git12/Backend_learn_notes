#Laravel 

- Laravelではイベントとリスナの仕組みが用意されている
- どこかのプログラムでイベントクラスを呼び出す(発火させる)と、リスナクラスの処理が実行されるという仕組み

- イベント(Event)
	- 何かが起きたという事実を表すクラス
- リスナ(Listener)
	- イベントを受け取って処理を行うクラス

- 使用例
	- 例えば「ユーザー登録が完了した」というイベントに対して
		- 「ウェルカムメールを送る」
		- 「ログを記録する」
		- 「ポイントを付与する」
	- の複数の処理をそれぞれ独立したリスナで実装できる
### 1. イベントクラスの作成
```bash
php artisan make:event UserRegistered
```

```php
// app/Events/UserRegistered.php
class UserRegistered
{
    use Dispatchable, InteractsWithSockets, SerializesModels;

    public function __construct(public User $user) {}
}
```
### 2. リスナクラスの作成
```bash
php artisan make:listener SendWelcomeMail --event=UserRegistered
```

```php
// app/Listeners/SendWelcomeMail.php
class SendWelcomeMail
{
    public function handle(UserRegistered $event): void
    {
        Mail::to($event->user)->send(new WelcomeMail());
    }
}
```

### 3. イベントとリスナの紐付け
`EventServiceProvider` に登録します。

```php
// app/Providers/EventServiceProvider.php
protected $listen = [
    UserRegistered::class => [
        SendWelcomeMail::class,
        RecordUserLog::class,    // 複数登録可能
        GrantInitialPoints::class,
    ],
];
```

### 4. イベントの発火

```php
// コントローラなどから
UserRegistered::dispatch($user);

// または
event(new UserRegistered($user));
```
