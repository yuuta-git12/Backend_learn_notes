- クライアントが生成したリクエストの内容をチェックすること
	- Laravelにはバリデーションの方法がいくつかある
		- `FormRequest`クラスを使う方法
- `app/Http/Requests`フォルダに作成される
- `authorized`メソッド：
	- ログイン中のユーザーが対象のレコードの更新権限を持っているかなどを確認するメソッド
- `rules`メソッド
	- パラメータの内容に不正がないか確認するためのメソッド
	- バリデーションルールを設定した連想配列を返す
		- キー：内容を確認したいパラメータの名前
		- 値：バリデーションルール
```php
// キー名も日本語に変更

'attributes' => [
'category_id' => 'カテゴリ',
'title' => 'タイトル',
'price' => '価格',
],
```

## バリデーションメッセージの日本語化
- バリデーションのエラーメッセージは以下で管理されている
	- `lang/en/validation.php`
- 以下のコマンドで`lang`フォルダを作成する必要がある
- 同時に`validation.php`も作成される
```
php artisan lang:publish	
```

- 日本語メッセージを作成する場合は以下の手順
	- `lang`に`ja`フォルダを作成
	- `ja`フォルダ内に`validation.php`を作成
	- `validation.php`内にエラーメッセージの連想配列を記載する
	- `config/app.php`の`locale`を`en`から`ja`に変更する



## 配列に対するバリデーション
- `キー名.*`とすることで`キー名`が配列なら、配列の要素に対するバリデーションを設定できる
- 例
```php
public function rules(): array
{
return [
// 配列user_idsの要素にバリデーションを実施、必須、userテーブルに存在するID
'user_ids.*' => 'required|exists:user,id',
];
}
```
