-  Laravelのファサードとは、**サービスコンテナ内のクラスへのシンプルで静的な呼び出しインターフェース**を提供する機能です。

- **簡単に言うと：** 複雑な依存性注入やインスタンス化をせずに、静的メソッドのように見える構文でLaravelの機能を使えるようにする仕組みです。

**例：**
```php
// ファサードを使う場合（シンプル）
use Illuminate\Support\Facades\Cache;
Cache::get('key');

// ファサードを使わない場合（複雑）
$cache = app()->make('cache');
$cache->get('key');
```

- **よく使われるファサード：**
	- `DB::` - データベース操作
	- `Cache::` - キャッシュ操作
	- `Route::` - ルート定義
	- `Auth::` - 認証
	- `Storage::` - ファイルストレージ
	- `Log::` - ログ記録

- **仕組み：** ファサードは実際には静的メソッドではなく、`__callStatic()` マジックメソッドを使って、サービスコンテナから対応するインスタンスを取得し、メソッドを動的に呼び出しています。

- 公式ドキュメント：[https://laravel.com/docs/facades](https://laravel.com/docs/facades)
