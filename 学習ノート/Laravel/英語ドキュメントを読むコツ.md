## よく出る技術英語の解説

日常英語との違いは「動詞の使い方」です。技術文書特有の動詞を覚えると一気に読みやすくなります。

**Laravelドキュメントで特によく出る動詞：**

|英語|意味|例文イメージ|
|---|---|---|
|retrieve|取得する|retrieve a user from the database|
|resolve|解決する・取り出す|resolve a class out of the container|
|bind|登録する|bind an interface to an implementation|
|dispatch|実行・送出する|dispatch a job to the queue|
|leverage|活用する|leverage Laravel's built-in features|
|scaffold|自動生成する|scaffold the login views|
|extend|拡張する|extend the base class|
|publish|公開・出力する|publish the config file|

**よく出る名詞・概念語：**

|英語|意味|
|---|---|
|under the hood|内部的には・裏側では|
|out of the box|デフォルトで・設定不要で|
|given|〜が与えられた場合（条件を示す）|
|subsequent|それ以降の・後続の|
|convention|慣習・規約|
|implementation|実装・具体的なクラス|
|overhead|余分なコスト・負荷|

## ドキュメントの読み方のコツ

**① コードブロックを先に読む**

Laravelのドキュメントは必ずコード例がセットです。コードを見てから説明文を読むと、「この英語はあの処理を説明しているんだ」と対応づけやすくなります。

**② セクションの最初の1文だけ読む**

技術文書は最初の1文に要点が凝縮されています。まず最初の文だけ読んで、必要なセクションかどうか判断する習慣をつけると効率的です。

**③ 「may」「can」「should」「must」の違いを意識する**

- `may` → ～することがある・～してもよい
- `can` → ～できる
- `should` → ～すべき（推奨）
- `must` → ～しなければならない（必須）

これを意識するだけで、「必須の設定」と「オプションの設定」の区別がつくようになります。

**④ 知らない単語は飛ばしてOK**

技術文書は同じ単語が何度も出ます。最初は意味がわからなくても、何度か見るうちに文脈で自然と覚えます。最初から全部理解しようとしないのがコツです。
