# PHP 標準クラスライブラリについて


## PHP
- 元々は潤沢なライブラリが特徴
- 近年のオブジェクト指向型プログラミング構成に伴い、PHPでもオブジェクト指向型のライブラリが多く提供されている

## 関数とクラスの違い
### 関数
- データは返すが、関数内には止まらない

### クラス
- 自身でデータを持つことができる
- 処理前または後のデータを保存できる
- 「データを操作するための機能（関数）を備えた高機能な器（変数）」

### クラスからインスタンスを作成した場合のメモリの扱い
- `new` キーワードでインスタンスを作成すると、ヒープメモリ上にオブジェクトが確保される
- インスタンスを格納した変数は、オブジェクトへの「参照（ハンドル）」を保持する
- 同じオブジェクトを別の変数に代入しても、デフォルトでは参照が共有される（同じメモリ領域を指す）
- 完全にコピーしたい場合は `clone` キーワードを使う
- インスタンスが不要になった場合は、PHPのガベージコレクタが自動的にメモリを解放する

```php
$obj1 = new MyClass();  // ヒープにオブジェクト確保
$obj2 = $obj1;          // 同じオブジェクトへの参照（コピーではない）
$obj3 = clone $obj1;    // 独立したコピーを作成
```

## オブジェクト変数
- クラスから `new` キーワードを使って生成された実体のことを**インスタンス（オブジェクト）**と呼ぶ
- インスタンスを格納する変数を**オブジェクト変数**という
- オブジェクト変数を通じて、クラスのプロパティやメソッドにアクセスする

```php
class Car {
    public string $color = 'red';

    public function drive(): string {
        return "走る";
    }
}

// インスタンスの生成
$car = new Car();  // $car がオブジェクト変数

// プロパティへのアクセス
echo $car->color;   // red

// メソッドの呼び出し
echo $car->drive(); // 走る
```

### コンストラクタ
- クラスからインスタンス生成時に自動的に呼ばれる特殊なメソッド
- 初期値のセットアップや依存関係の注入に使う

```php
class User {
    public string $name;
    public int $age;

    public function __construct(string $name, int $age) {
        $this->name = $name;
        $this->age  = $age;
    }
}

$user = new User('田中', 25);
echo $user->name; // 田中
echo $user->age;  // 25
```

### PHP8 のコンストラクタ・プロモーション
- コンストラクタの引数に `public` / `protected` / `private` を付けるだけでプロパティ宣言と代入を同時に行える

```php
class User {
    public function __construct(
        public string $name,
        public int $age
    ) {}
}
```

## メンバー関数 / メンバー変数（メソッド / プロパティ）

### プロパティ（メンバー変数）
- クラスが保持するデータ（状態）を表す変数
- アクセス修飾子で外部からのアクセスを制御する

| 修飾子      | アクセス範囲                          |
|-------------|---------------------------------------|
| `public`    | どこからでもアクセス可                 |
| `protected` | 自クラスと継承クラスのみ              |
| `private`   | 自クラスのみ                          |

```php
class BankAccount {
    public string $owner;       // 外部から読み書き可
    protected float $balance;  // 継承先からもアクセス可
    private string $pin;       // このクラス内のみ

    public function __construct(string $owner, float $balance, string $pin) {
        $this->owner   = $owner;
        $this->balance = $balance;
        $this->pin     = $pin;
    }
}
```

### メソッド（メンバー関数）
- クラスに定義された関数
- クラスのプロパティへは `$this->プロパティ名` でアクセスする

```php
class BankAccount {
    private float $balance;

    public function __construct(float $balance) {
        $this->balance = $balance;
    }

    public function deposit(float $amount): void {
        $this->balance += $amount;
    }

    public function getBalance(): float {
        return $this->balance;
    }
}

$account = new BankAccount(1000);
$account->deposit(500);
echo $account->getBalance(); // 1500
```

## null 安全演算子（PHP8）
- `?->` 演算子を使うと、オブジェクトが `null` でも安全にメソッドやプロパティにアクセスできる
- `null` の場合は処理をスキップして `null` を返す（エラーにならない）

### PHP8 以前の書き方
```php
// null チェックが必要で冗長になりやすい
$city = null;
if ($user !== null) {
    $address = $user->getAddress();
    if ($address !== null) {
        $city = $address->getCity();
    }
}
```

### PHP8 以降（null 安全演算子）
```php
// ?-> を使うとシンプルに書ける
$city = $user?->getAddress()?->getCity();
// $user が null なら $city は null になる（エラーなし）
```

### ポイント
- チェーンの途中で `null` になった時点でそれ以降の評価はスキップされる
- 代入の左辺には使えない（読み取り専用）

## 静的メソッド / 静的プロパティ（クラスメソッド / クラスプロパティ）
- `static` キーワードを付けると、**インスタンスを生成せずに**クラス名から直接呼び出せる
- `self::` で自クラスの静的メンバーにアクセスする
- `::` はスコープ定義演算子（Scope Resolution Operator）と呼ばれる

### 静的プロパティ
```php
class Counter {
    public static int $count = 0;  // クラス全体で共有される変数
}

Counter::$count++;
Counter::$count++;
echo Counter::$count; // 2
```

### 静的メソッド
```php
class MathHelper {
    public static function square(int $n): int {
        return $n * $n;
    }
}

// インスタンス不要で呼び出せる
echo MathHelper::square(5); // 25
```

### static vs self
| キーワード | 意味                                         |
|------------|----------------------------------------------|
| `self::`   | 定義されたクラス自身を参照する               |
| `static::` | 実行時のクラス（遅延静的束縛）を参照する     |

```php
class ParentClass {
    public static function create(): static {
        return new static(); // 呼び出し元のクラスでインスタンス化
    }
}

class ChildClass extends ParentClass {}

$child = ChildClass::create(); // ChildClass のインスタンスが返る
```

### 用途の目安
- **静的メソッド**：ファクトリメソッド、ユーティリティ関数、設定値の取得など
- **静的プロパティ**：アプリ全体で共有するカウンターや設定値など
- インスタンスの状態に依存する処理は通常のメソッドを使う

## 主な PHP 標準クラスライブラリ

### DateTime / DateTimeImmutable
- 日付・時刻を扱うクラス
- `DateTimeImmutable` は変更不可（メソッドが新しいインスタンスを返す）

```php
$dt = new DateTimeImmutable('2026-03-01');
$next = $dt->modify('+7 days');
echo $next->format('Y-m-d'); // 2026-03-08
```

### Exception / RuntimeException
- 例外処理の基底クラス
- `try / catch / finally` と組み合わせて使う

```php
function divide(int $a, int $b): float {
    if ($b === 0) {
        throw new RuntimeException('ゼロ除算エラー');
    }
    return $a / $b;
}

try {
    echo divide(10, 0);
} catch (RuntimeException $e) {
    echo $e->getMessage(); // ゼロ除算エラー
}
```

### ArrayObject
- 配列をオブジェクトとして扱えるクラス
- `IteratorAggregate` を実装しており `foreach` で反復可能

```php
$ao = new ArrayObject(['a', 'b', 'c']);
$ao->append('d');
echo $ao->count(); // 4
```

### SplStack / SplQueue
- SPL（Standard PHP Library）が提供するスタック・キュー構造

```php
$stack = new SplStack();
$stack->push('first');
$stack->push('second');
echo $stack->pop(); // second（後入れ先出し）

$queue = new SplQueue();
$queue->enqueue('first');
$queue->enqueue('second');
echo $queue->dequeue(); // first（先入れ先出し）
```
