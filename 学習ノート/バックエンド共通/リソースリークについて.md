**リソースリーク（resource leak）**とは、
プログラムが確保した「リソース」を、使い終わったあとに正しく解放しないことです。

🔎 リソースって何？

メモリ以外にもいろいろあります。

メモリ

データベース接続

ファイルハンドル

ソケット

ロック

スレッド

S3クライアントなどの外部接続

💥 何が問題なの？

解放しないまま処理を続けると…

メモリ不足になる

DB接続上限に達する

ファイルが開きっぱなしになる

アプリが重くなる／落ちる

長時間稼働バッチで致命傷になる

🧠 例（PHP）
❌ リークする例
$result = $DB->query($sql);
return $result;  // freeしていない

結果セットが解放されないままになります。

✅ 正しい例
$result = $DB->query($sql);
$data = $result->fetchAll();
$result->free();   // 解放
return $data;

※ あなたが前に言っていた
returnの前に$DB->free()を入れると配列が空になる
というのは、「結果を読み切る前に解放している」可能性が高いです。

🏗 よくある原因

例外発生時にcloseし忘れる

returnの前で解放していない

ループ内で接続を作りっぱなし

finallyを書いていない

🛠 対策パターン
① try-finally
$result = $DB->query($sql);
try {
    return $result->fetchAll();
} finally {
    $result->free();
}
② スコープに任せる（PDOなど）

PDOはスクリプト終了時に自動解放されます。

🎯 あなたのケース（DBまわり）

今やっているような

PDF用データ取得

DB→配列変換

その後free

という流れは正しいです。

大事なのは：

✅ 「必要なデータを全部取得してから解放する」

🔥 まとめ
リーク対象	影響
メモリ	OOM
DB接続	接続枯渇
ファイル	OS制限超過
ロック	デッドロック
