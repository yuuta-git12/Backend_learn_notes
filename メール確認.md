# Laravel メール確認（メールアドレス認証）

---

## MustVerifyEmail

- `implements`：インターフェースを実装する際に必要な記述
- `MustVerifyEmail` は Laravel 標準のメールアドレス認証インターフェース
- `User` モデルに実装するだけで、メール認証機能が有効になる
- 認証前のユーザーが保護されたページにアクセスすると、確認メールの送信ページにリダイレクトされる

### User モデルへの実装

```php
// app/Models/User.php
use Illuminate\Contracts\Auth\MustVerifyEmail;

class User extends Authenticatable implements MustVerifyEmail
{
    // ...
}
```

### インターフェースが要求するメソッド

`MustVerifyEmail` を `implements` するとLaravel が以下の実装を自動的に提供する（`Illuminate\Auth\MustVerifyEmail` トレイト経由）。

| メソッド                              | 説明                                       |
|---------------------------------------|--------------------------------------------|
| `hasVerifiedEmail(): bool`            | メールが認証済みかどうかを確認する         |
| `markEmailAsVerified(): bool`         | メールを認証済みとしてDBに記録する         |
| `sendEmailVerificationNotification()` | 確認メールを送信する                       |
| `getEmailForVerification(): string`   | 認証対象のメールアドレスを返す             |

### ルーティングの設定

```php
// routes/web.php
use Illuminate\Support\Facades\Route;

// メール認証関連のルートを有効化
Auth::routes(['verify' => true]);

// または個別に定義する場合
Route::get('/email/verify', function () {
    return view('auth.verify-email');
})->middleware('auth')->name('verification.notice');

Route::get('/email/verify/{id}/{hash}', function (EmailVerificationRequest $request) {
    $request->fulfill();
    return redirect('/home');
})->middleware(['auth', 'signed'])->name('verification.verify');

Route::post('/email/verification-notification', function (Request $request) {
    $request->user()->sendEmailVerificationNotification();
    return back()->with('message', '確認メールを再送しました');
})->middleware(['auth', 'throttle:6,1'])->name('verification.send');
```

### `verified` ミドルウェア

- メールが認証済みのユーザーのみアクセスを許可するミドルウェア
- 未認証の場合は `verification.notice` ルートへリダイレクトされる

```php
// 特定のルートを認証済みユーザーのみに制限
Route::get('/dashboard', function () {
    return view('dashboard');
})->middleware(['auth', 'verified']);

// ルートグループでまとめて適用
Route::middleware(['auth', 'verified'])->group(function () {
    Route::get('/profile', [ProfileController::class, 'show']);
    Route::get('/settings', [SettingsController::class, 'index']);
});
```

### メール認証の流れ

```
1. ユーザー登録
      ↓
2. 確認メールが自動送信される（sendEmailVerificationNotification）
      ↓
3. ユーザーがメール内のリンクをクリック
      ↓
4. markEmailAsVerified() が呼ばれ、email_verified_at に日時が記録される
      ↓
5. verified ミドルウェアで保護されたページにアクセス可能になる
```

### `users` テーブルの `email_verified_at` カラム

- 認証完了時に日時が記録される
- `null` の場合は未認証

```php
// 認証状態の確認
if ($user->hasVerifiedEmail()) {
    // 認証済み
}

// 手動で認証済みにする（管理画面などで使用）
$user->markEmailAsVerified();
```

## Mailpit

- `.env` にメーラーやホストを設定する記述が存在する
- ローカル開発環境用の**メールキャッチャーツール**
- アプリから送信されたメールを実際には外部に送らず、Mailpit が受け取って Web UI で確認できる
- メール送信機能のローカルテストに使用する

### .env の設定

```env
MAIL_MAILER=smtp
MAIL_HOST=localhost
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"
```

> Docker 環境の場合は `MAIL_HOST=mailpit`（サービス名）にする。

### Docker Compose での設定

```yaml
# docker-compose.yml
services:
  app:
    # ...

  mailpit:
    image: axllent/mailpit
    ports:
      - "1025:1025"   # SMTP ポート（アプリからの送信を受け取る）
      - "8025:8025"   # Web UI ポート（ブラウザで確認）
```

```env
# Docker 環境の .env
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
```

### Web UI でのメール確認

- ブラウザで `http://localhost:8025` にアクセスする
- 受信したメールの一覧・本文・HTMLプレビュー・ヘッダー情報を確認できる
- メール確認リンクも UI 上からそのままクリックできる

### よく使う確認の流れ

```
1. ユーザー登録 or パスワードリセットをアプリで実行
      ↓
2. http://localhost:8025 にアクセス
      ↓
3. 受信した確認メールを開く
      ↓
4. メール内のリンクをクリックして認証完了
```
